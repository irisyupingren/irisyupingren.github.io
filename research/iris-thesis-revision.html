1. To discuss

1.1. Rework contribution

The five bullet point, 2-3 sentences each version:

  • This dissertation brings together perspectives from musical pattern discovery, functional programming, and other relevant areas of research, to form an all-around view of state-of-the-art pattern discovery algorithms. The richness of the concept of pattern and transformation are shown and a connection between the two concepts is made.
  • This dissertation puts forward novel annotation tools in order to gather new musical pattern annotation datasets. The dissertation presents investigations and comparisons of the annotation datasets using a newly devised inter-annotator agreement measure. It is shown that factors such as the musical background of the annotators and the interfaces of the annotation tools can influence how much annotators agree with each other.
  • New methods to evaluate musical pattern discovery algorithms are proposed. This dissertation adopts methods consisting of visualisation, fusion, classification, synthetic data, and prediction. This dissertation advocates for a diverse range of evaluation methodologies for algorithms as the applications for musical pattern discovery algorithms are equally diverse.
  • Using the functional programming language, Haskell, this dissertation makes use of concepts from category theory to create a DSL for analysing musical patterns. The DSL implements the connection between musical patterns and transformations, and provides the ability to quantitatively analyse groups of transformations that an analyst specifies.
  • With the analysis and the DSL presented in this dissertation, two decisive differences between human-annotated patterns and algorithmically extracted patterns are presented. The first difference is in rhythmic features. The second difference is the composition of transformations between pattern occurrences.

Using "we":

  • We bring together perspectives from musical pattern discovery, functional programming, and other relevant areas of research, to form an all-round view on state-of-the-art pattern discovery algorithms. We show the richness of the concept of pattern and transformation, and make connection between the two concepts.
  • We devise novel annotation tools in order to gather new musical pattern annotation datasets. We investigate and compare the datasets using an inter-annotator agreement measure we devise. We show that factors such as the musical background of the annotators and the interfaces of the annotation tools can influence how much annotators agree with each other.
  • We propose new methods to evaluate musical pattern discovery algorithms. Our methods consists of visualisation, fusion, classification, synthetic data, and prediction. We advocate for a diverse range of evaluation methodologies for algorithms as the applications for musical pattern discovery algorithms are equally diverse.
  • Using the functional programming language, Haskell, we use concepts from category theory to create a DSL for analysing musical patterns. Our DSL implements the connection between musical patterns and transformations, and provides us with the ability to quantitatively analyse groups of transformations that we specify.
  • Using the DSL we implement, we find two decisive differences between human-annotated patterns and algorithmically extracted patterns. The first difference being rhythmic features. The second difference being the composition of transformations between pattern occurrences.

4.1.2. In order to be able to indicate your main contributions in the introduction, you need to specify the “Grand questions” you have mentioned in the introduction and arrive at more specific questions which one can investigate within the scope of a thesis.

“Are we, as a species, united in how we perceive, understand, and create patterns? Is pattern finding a mechanical process that can be automated using machines? This dissertation aims to provide evidence, analysis, and tools to address these questions in the domain of music.”

You cannot answer the first question in your PhD thesis (too big). The second question: how could a potential answer look like? There are lots of automatic algorithms on the topic already, so what do you mean by “can be automated using machines”? Do you mean an exact replication of what humans do?

I did not promise that I'll answer them. I'm using them to set the scene and pique interests. I made it clearer that the questions motivate this dissertation and we only tap into a very small part.

“Patterns, as a type of highly subjective and ubiquitous abstraction, have important connections to transformations, which can be explored com- positionally using a functional programming language to reveal implications for designing musical pattern discovery algorithms. We unpack this sentence into three main aspects—understanding patterns, making connections to transformations, and using a functional programming language. “

This is a high-level description of what you do in the thesis. It is good to start from a high-level description, but you also need to arrive at specific descriptions and questions for your dissertation in order to communicate your specific scientific contribution in section 1.8.
